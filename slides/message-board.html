jQuery

表單送出：攔截 submit + AJAX

$(function () {
  const $form = $('#message-form');
  const $name = $('#name');
  const $content = $('#content');

  $form.on('submit', function (event) {
    event.preventDefault();

    $.ajax({
      url: 'api/create.php',
      method: 'POST',
      dataType: 'json',
      data: {
        name: $.trim($name.val()),
        content: $.trim($content.val())
      }
    }).done(function () {
      $content.val('');
      loadMessages();
    });
  });
});

重點

  • $(function(){})：DOM ready 後初始化。
  • event.preventDefault()：避免表單傳統跳頁。
  • $.ajax()：非同步呼叫 PHP。
  • 成功後清空輸入並重新載入列表。

jQuery

列表載入：GET API + render

function loadMessages() {
  return $.ajax({
    url: 'api/list.php',
    method: 'GET',
    dataType: 'json'
  }).done(function (response) {
    if (response && response.success) {
      renderMessages(response.messages || []);
    }
  });
}

function escapeHtml(value) {
  return $('<div>')
    .text(value == null ? '' : String(value))
    .html();
}

實作亮點

  • 新增與讀取都走 API，畫面不需要刷新。
  • 使用 .text() 做 escape，降低 XSS 風險。
  • 留言以最新時間排序後顯示。

PHP

新增留言 API：驗證 request

<?php
header('Content-Type: application/json; charset=utf-8');

if ($_SERVER['REQUEST_METHOD'] !== 'POST') {
    http_response_code(405);
    echo json_encode([
        'success' => false,
        'error' => '只接受 POST 請求。',
    ]);
    exit;
}

$name = trim((string)($_POST['name'] ?? ''));
$content = trim((string)($_POST['content'] ?? ''));

if ($name === '' || $content === '') {
    http_response_code(422);
    // 回傳欄位驗證錯誤
}

重點

  • PHP 負責接收 $_POST。
  • 先驗證 method 與欄位內容。
  • 錯誤回 HTTP status code + JSON。

PHP + PDO

資料庫寫入：Prepared Statement

$stmt = $pdo->prepare(
    'INSERT INTO messages (name, content, created_at)
     VALUES (:name, :content, NOW())'
);

$stmt->execute([
    ':name' => $name,
    ':content' => $content,
]);

$id = (int)$pdo->lastInsertId();

http_response_code(201);
echo json_encode([
    'success' => true,
    'message' => $message,

實作亮點

  • 不把使用者輸入直接串進 SQL。
  • :name / :content 由 PDO 綁定。
  • 成功新增回傳 201 Created。

API Contract

統一 JSON 回應格式

// 成功
{
  "success": true,
  "messages": [
    {
      "id": 1,
      "name": "Joe",
      "content": "Hello",
      "created_at": "2026-05-17 12:00:00"
    }
  ]
}

// 失敗
{
  "success": false,
  "error": "留言內容必填。"
}

設計理由

  • 前端只要判斷 success。
  • 錯誤訊息固定放在 error。
  • 後續加刪除、編輯、分頁時可沿用。

Schema

Table Schema：messages

欄位型別用途
idINT AUTO_INCREMENT每則留言的唯一識別。
nameVARCHAR(30)留言者暱稱；最多 30 字。
contentTEXT留言內容；最多 500 字。
created_atDATETIME建立時間；用於排序與顯示。

SQL

建表 SQL

CREATE DATABASE IF NOT EXISTS message_board
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE message_board;

CREATE TABLE IF NOT EXISTS messages (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(30) NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

不指定 ENGINE=InnoDB，讓 XAMPP 與 Endor LAMP Sandbox 都能使用環境預設 storage engine。

Security

安全性與輸入驗證

SQL Injection

使用 PDO prepared statement，不直接拼接 SQL。

XSS

前端 render 前 escape 使用者輸入，script tag 只會顯示為文字。

Validation

空欄位與超長欄位回傳 HTTP 422，避免壞資料進資料庫。
