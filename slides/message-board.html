留言板專案

這份簡報用課堂報告的角度，說明我如何用 jQuery、PHP、MySQL 與 XAMPP-compatible 環境做出一個單頁留言板。

jQuery DOM / AJAX PHP 表單處理 XAMPP / LAMP 部署 MySQL Table Schema

Project Goal

這個程式在做什麼？

新增留言

使用者輸入暱稱與留言內容，前端不重新整理頁面，透過 AJAX POST 到 api/create.php。

讀取列表

頁面載入或按「重新載入」時，呼叫 api/list.php 取得最新留言。

基本安全

後端驗證欄位長度，SQL 使用 prepared statements；前端 render 前 escape 內容避免 XSS。

Class Focus

這門課學到的東西，我怎麼講給同學聽？

jQuery

用 $(function(){ ... }) 等 DOM ready 後綁定事件；用 $.ajax() 非同步呼叫 PHP API，不需要重新整理頁面。

PHP

用 $_POST 接收表單資料，做必填/長度驗證，再用 PDO 連線資料庫並回傳 JSON。

XAMPP / LAMP

把專案放進 htdocs，啟動 Apache + MySQL，匯入 schema 後就可以用 browser 測試完整流程。

jQuery Implementation

前端：用 jQuery 做事件處理與 AJAX

$(function () {
  const $form = $('#message-form');
  const $name = $('#name');
  const $content = $('#content');

  $form.on('submit', function (event) {
    event.preventDefault();

    $.ajax({
      url: 'api/create.php',
      method: 'POST',
      dataType: 'json',
      data: {
        name: $.trim($name.val()),
        content: $.trim($content.val())
      }
    }).done(function () {
      $content.val('');
      loadMessages();
    });
  });
});

我想強調的 jQuery 重點

  • $(function(){})：等 DOM 載入後再操作元素。
  • event.preventDefault()：阻止表單傳統跳頁送出。
  • $.ajax()：把資料送到 PHP，並期待 JSON 回應。
  • .done() / .fail()：分開處理成功與失敗。

Design Highlight

亮點設計 1：不刷新頁面的留言列表

function loadMessages() {
  return $.ajax({
    url: 'api/list.php',
    method: 'GET',
    dataType: 'json'
  }).done(function (response) {
    if (response && response.success) {
      renderMessages(response.messages || []);
    }
  });
}

function escapeHtml(value) {
  return $('<div>')
    .text(value == null ? '' : String(value))
    .html();
}

為什麼這樣設計？

  • 留言列表跟新增留言都走 API，前端只負責畫面更新。
  • 新增成功後呼叫 loadMessages()，畫面馬上同步最新資料。
  • 用 jQuery .text() escape，再放進 HTML，避免 demo 時被 XSS payload 亂掉。

Backend API

後端 API 設計

GET api/list.php

  • 連線資料庫。
  • 查詢 messages。
  • 排序：created_at DESC, id DESC。
  • 回傳 JSON：{success, messages}。

POST api/create.php

  • 只接受 POST。
  • 驗證暱稱必填、最多 30 字。
  • 驗證留言必填、最多 500 字。
  • 使用 PDO prepared statement 寫入。
  • 成功回傳 HTTP 201 與新增資料。

PHP Implementation

後端：PHP 驗證資料後寫入 MySQL

$name = trim((string)($_POST['name'] ?? ''));
$content = trim((string)($_POST['content'] ?? ''));

if ($name === '') {
    http_response_code(422);
    echo json_encode([
        'success' => false,
        'error' => '暱稱必填。',
    ]);
    exit;
}

$stmt = $pdo->prepare(
    'INSERT INTO messages (name, content, created_at)
     VALUES (:name, :content, NOW())'
);
$stmt->execute([
    ':name' => $name,
    ':content' => $content,
]);

我在 PHP 這邊做了什麼？

  • 用 $_POST 接收表單送來的資料。
  • 用 trim() 清掉前後空白。
  • 錯誤時回傳 HTTP 422 與 JSON，前端可以顯示錯誤。
  • 寫入資料庫時用 prepared statement，避免 SQL Injection。

Design Highlight

亮點設計 2：API 回傳統一 JSON 格式

// 成功
{
  "success": true,
  "messages": [...]
}

// 失敗
{
  "success": false,
  "error": "留言內容必填。"
}

對課堂報告好講的地方

  • 前端不用猜 PHP 輸出格式，只要看 success。
  • 錯誤訊息由後端決定，前端負責顯示。
  • 這種格式之後要加刪除、編輯、分頁都可以沿用。

Database

Table Schema：messages

欄位 型別 限制 / 說明
id INT Primary Key、Auto Increment，用來唯一識別每一則留言。
name VARCHAR(30) 留言者暱稱；後端限制最多 30 字。
content TEXT 留言內容；後端限制最多 500 字。
created_at DATETIME 預設 CURRENT_TIMESTAMP，用於顯示與排序。

Schema 不強制指定 ENGINE=InnoDB，讓 XAMPP 與 Endor LAMP Sandbox 都可以使用環境預設 storage engine。

SQL

建表 SQL

CREATE DATABASE IF NOT EXISTS message_board
  CHARACTER SET utf8mb4
  COLLATE utf8mb4_unicode_ci;

USE message_board;

CREATE TABLE IF NOT EXISTS messages (
  id INT AUTO_INCREMENT PRIMARY KEY,
  name VARCHAR(30) NOT NULL,
  content TEXT NOT NULL,
  created_at DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP
) DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;

Security

安全性處理

SQL Injection

新增留言使用 PDO prepare() 與 :name、:content placeholders，不把使用者輸入直接串到 SQL。

XSS

前端 render 前使用 jQuery .text() escape 使用者內容，<script> 會顯示成文字而非執行。

Input Validation

後端回傳 HTTP 422 處理空欄位與超長欄位，避免無效資料進入資料庫。

XAMPP

XAMPP 在這個專案裡的角色

它提供了什麼？

  • Apache：負責把 index.html、CSS、JS、PHP API 變成瀏覽器可開的網站。
  • PHP：執行 api/create.php、api/list.php。
  • MySQL/MariaDB：保存留言資料。
  • phpMyAdmin：用圖形介面匯入 schema.sql。
# 專案放置位置
C:\xampp\htdocs\message-board\

# 匯入資料庫
mysql -u root < database/schema.sql

# 瀏覽器開啟
http://localhost/message-board/

Deployment

XAMPP / Endor 部署重點

XAMPP

  • 放到 htdocs/message-board/。
  • 啟動 Apache 與 MySQL。
  • 匯入 database/schema.sql。
  • 開啟 http://localhost/message-board/。

Endor LAMP Sandbox

  • 掛到 /var/www/localhost/htdocs/app。
  • MariaDB 帳密為 root/root。
  • 匯入 schema 後開 /app/。
  • 若 private repo mount 失敗，需改 public 或用 token 下載。
